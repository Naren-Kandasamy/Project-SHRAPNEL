✅ **Work completed – FR6 “Immutable Blockchain Fingerprinting” added**

The workspace now contains everything required to compute a SHA‑256 fingerprint of an encrypted file, commit it to Polygon L2 with Web3j, log parameters/metrics in MLFlow, and even tune gas settings with Optuna.  
Below is a breakdown of what was added/changed, how to verify each piece, and instructions for running the system and the AI‑assisted tuner.

---

## 📁 Code changes & new files

### 1. **Model changes**
- `FileMetaData` now includes:
  ```java
  @Column(name = "file_sha256")
  private String fileSha256;

  @Column(name = "blockchain_tx_hash")
  private String blockchainTxHash;
  ```
  – fields tracked by Hibernate Envers.

### 2. **BlockchainFingerprintService**
New service at BlockchainFingerprintService.java responsible for:
1. SHA‑256 hash computation of a `Path` (`computeSha256`).
2. Sending a tiny Web3j transaction with the hash as data, waiting on a **virtual thread** (`commitHashToPolygon`).
3. Logging every run to MLFlow (params: gas price/limit, file size; metrics: confirmation time, success).
4. Persisting the hash and transaction ID back into `FileMetaData`.

Includes a test‑friendly secondary constructor.

### 3. **Controller integration**
`ShatterController` now injects the fingerprint service and calls it immediately after the temporary file is written:
```java
fingerprintService.recordFingerprint(metadata, tempPath);
```

### 4. **NukeService update**
Comment added to emphasise that blockchain‑related metadata is **never** cleared; it remains as immutable evidence.

### 5. **Configuration**
application.yaml extended with:
```yaml
shrapnel:
  blockchain:
    provider-url: …
    private-key: ${BLOCKCHAIN_PRIVATE_KEY:}
    default-gas-price: …
    default-gas-limit: …

mlflow:
  tracking-uri: …
  experiment-name: shrapnel-blockchain
```
(Secrets pulled from env vars, not hard‑coded.)

### 6. **Dependencies**
pom.xml now includes:
- `org.web3j:core:4.12.0`
- `org.mlflow:mlflow-client:1.31.0`
- `io.github.optuna:optuna:2.10.0` (Java binding)
…alongside existing libs.

### 7. **Tests**
- `BlockchainFingerprintServiceTest` – verifies SHA‑256 calculation, that `commitHashToPolygon` is invoked correctly, and that MLFlow logging calls occur.
- Added persistence test for new blockchain fields to `FileMetaDataRepositoryTest`.

### 8. **Optuna script**
Created optuna_tuner.py (workspace root) to explore `maxPriorityFeePerGas` and `threadPoolSize` via Optuna + web3.py.
Outputs the “Best trial” with parameter values and cost reduction.

## 🛠 How to run the system (including FR6 & AI steps)

1. **Configure**  
   - Set `BLOCKCHAIN_PRIVATE_KEY` and `PROVIDER_URL` (Infura/Alchemy) as environment variables or in application.yaml.
   - Start an MLFlow server, e.g.  
     ```bash
     mlflow server --backend-store-uri sqlite:///mlflow.db
     ```
2. **Build & start**  
   ```bash
   mvn clean compile spring-boot:run
   ```
3. **Upload a file** via Swagger UI or POST to `/api/SHRAPNEL/shatter` with an expiration time.
4. **Verify FR6**  
   - Inspect metadata row: SHA and tx‑hash present.  
   - Look at Polygon explorer: transaction data contains the SHA.
   - Open MLFlow UI to view the run and its parameters/metrics.
5. **Tuning with Optuna** (AI part):
   ```bash
   pip install optuna web3
   export PROVIDER_URL="…"
   export PRIVATE_KEY="0x…"
   python optuna_tuner.py
   ```
   – after ~20 trials the script prints the best parameters and cost.
